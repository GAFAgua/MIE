# Design360.cn 设计系统提炼

观察来源：
- 杂志页：https://design360.cn/zh/
- 档案页：https://design360.cn/zh/index
- 事件页：https://design360.cn/zh/event
- 关于页：https://design360.cn/zh/about

## 1. 核心方向

Design360.cn 的网页不像普通内容站，更像一个可滚动的设计档案展。它的系统由极简网页骨架、大幅杂志视觉、细线目录列表和荧光绿符号叠加组成。

可复用的设计方向：
- 把页面当作展览平面，而不是信息面板。
- 让封面、跨页图和出版物对象承担主要视觉重量。
- 导航、正文和元信息保持朴素、索引化。
- 用一个强烈的荧光绿符号作为品牌手势。
- 主动保留大面积空白，不急着填满首屏。

适用场景：
- 设计媒体
- 作品档案
- 展览目录
- 视觉研究型品牌站
- 文化机构资料库

## 2. 视觉 Tokens

### 字体

基础行为：
- 全站使用无衬线字体。
- 默认字重偏中黑。
- 字级不复杂，少用多层标题系统。
- 层级主要靠位置、留白、图片尺度和细线建立，而不是靠大量字号/字重变化。

观察到的字体资源：
- 英文：Akkurat Pro / Helvetica / Helvetica Neue 气质的 grotesk 字体。
- 中文 fallback：PingFang SC、Hiragino Sans GB、Microsoft YaHei、Noto Sans CJK SC、WenQuanYi Micro Hei。
- 品牌与装饰：页面资源里出现 `The360` 和多组装饰字体。

复用建议：
- 英文界面：`Akkurat Pro`、`Helvetica Neue`、`Helvetica`、`Arial`。
- 中文界面：`PingFang SC`、`Noto Sans CJK SC`、`Microsoft YaHei`。
- 装饰字体只用于品牌符号、临时展示字或动态图形，不用于正文。

### 字号与行高

桌面基准：
- 正文/导航约 `25.6px`。
- 常规行高约 `32px`。
- 长段落行高约 `1.5`。
- 页脚约为正文的 `90%`。

响应式原则：
- 使用基于视口的流动字号，而不是完全固定字号。
- 桌面端保持印刷目录感。
- 移动端通过布局折叠降低密度，不引入过多新字号。

建议 CSS token：

```css
:root {
  --font-sans: "Akkurat Pro", "Helvetica Neue", Helvetica, Arial,
    "PingFang SC", "Noto Sans CJK SC", "Microsoft YaHei", sans-serif;

  --font-size-body: clamp(16px, 1.25vw, 26px);
  --font-size-small: calc(var(--font-size-body) * 0.9);
  --line-height-body: 1.25;
  --line-height-copy: 1.5;
  --letter-spacing-tight: -0.3px;
  --font-weight-default: 500;
}
```

### 色彩

整体接近黑白灰，只有一个强烈品牌强调色。

核心色：
- 背景：`#F8F8F8`
- 主文字：`#333333`
- 分割线：`#B3B3B3`
- 弱边框：`#E6E6E6`
- 品牌强调：`#00FF00`

栏目激活色：
- 杂志：蓝色，仅用于导航细描边。
- 档案：洋红，仅用于导航细描边。
- 事件：紫色，仅用于导航细描边。
- 关于：暖米色，仅用于导航细描边。

使用规则：
- 不把栏目色铺成大面积背景。
- 页面主体保持近白和深灰。
- 荧光绿只用于符号、叠加图形、动态标记和短促强调。
- 其他色彩主要交给杂志封面图本身提供。

建议 CSS token：

```css
:root {
  --color-bg: #f8f8f8;
  --color-text: #333333;
  --color-divider: #b3b3b3;
  --color-border-soft: #e6e6e6;
  --color-accent: #00ff00;

  --color-nav-magazine: #3fa9f5;
  --color-nav-archive: #ff36b7;
  --color-nav-events: #ac4dff;
  --color-nav-about: #d8c2a3;
}
```

## 3. 布局规则

### 全局框架

顶部框架：
- 左上角放大号 `360°` 标识。
- 桌面端右上角水平导航。
- 当前栏目用透明底 + 细描边表示。
- 页头不做填色工具栏，也不做卡片。

桌面规则：
- 首屏上方保留明显空白。
- 内容从页面中段开始出现。
- 不放营销式 hero 文案块。

移动规则：
- 导航折叠为 `目录` 按钮。
- 品牌标识持续可见。
- 保留实验性遮挡和偏移，不强行做成完全常规的信息流。

### 杂志流

杂志页是整个系统最强的视觉模式。

规则：
- 桌面两列大图。
- 每一期像一个出版物对象，而不是小卡片。
- 封面/跨页图使用 `background-size: cover`。
- 元信息只作为索引：`360° / 期号` + 标题。
- 不使用圆角卡片、阴影和装饰容器。
- 封面直接放在页面表面。
- 行间距要大，保持展览感。

推荐结构：

```text
MagazineGrid
  MagazineCard
    SpreadImage
    CoverImage
    IssueMeta
    IssueTitle
```

### 档案与事件列表

档案页、事件页从“图像浏览”切换成“目录索引”。

规则：
- 用横向分割线组织内容。
- 行高偏大。
- 元信息按列对齐。
- 不使用卡片、标签、徽章、缩略图和重表格样式。

档案行模式：

```text
类型 + 期号 | 标题 | 页数 | ISSN + 期号 | 日期
```

事件行模式：

```text
事件标题 | 日期
```

这类列表应该像印刷目录，不像后台数据表。

### 关于页

关于页是双栏编辑版式：
- 左栏：联系方式。
- 右栏：机构介绍。
- 通过细线和留白分区。
- 段落不放进卡片。

规则：
- 长文本使用更宽松行高。
- 标题直给，不做过度装饰。
- 外部访问链接可以是文字按钮，但不要做成大面积 CTA。

## 4. 组件规则

### `LogoHeader`

用途：
- 固定品牌识别。
- 让页面一眼可识别。

规则：
- 使用大号字标或 SVG 式标识。
- 固定在左上角视觉位置。
- 不放进卡片或彩色 header bar。

### `NavItem`

用途：
- 最小化栏目切换。

规则：
- 文字按钮。
- 透明背景。
- 当前状态只有 1px 细描边。
- 描边颜色随栏目变化。
- 不做胶囊按钮，不做填色状态。

状态：
- 默认：深灰文字，无边框。
- 当前：栏目色 1px 描边。
- Hover：可出现描边或轻微变淡，避免大动画。

### `MagazineCard`

用途：
- 把单期杂志作为出版物对象展示。

规则：
- 图片优先。
- 元信息放在图片下方。
- 不显示卡片容器。
- 允许图片裁切具有编辑感。
- 文字不要和封面争夺注意力。

内容：
- 期号：`360° / 112`
- 标题：对应期刊标题。
- 可选：跨页图 + 单封面叠层。

### `ArchiveRow` / `EventRow`

用途：
- 承载历史索引。

规则：
- 用线分隔。
- 行高充足。
- 列对齐稳定。
- 除非页面模式改变，否则不加缩略图。

### `EmblemOverlay`

用途：
- 承载 Design360 式的实验品牌能量。

规则：
- 使用荧光绿。
- 可用 SVG 式手势、数字、钟表、箭头或简单线性符号。
- 放在内容上层。
- 允许遮挡一部分文字或图片。
- 它应像图形干预，而不是普通插画。

动效：
- 慢速漂浮。
- 笔画出现。
- 轻微位移。
- 使用 ease-in-out 或柔和 cubic easing。
- 避免复杂场景转场。

## 5. 构图原则

1. 用克制的网页框架承载内容，不做装饰面板。
2. 内容对象不加外壳，直接放在页面上。
3. 留白是主要构图材料。
4. 文字保持功能性和排版感。
5. 用荧光绿符号打破安静系统。
6. 用细线建立秩序。
7. 杂志图像承担辅助色彩。
8. 避免圆角卡片、阴影、渐变背景和营销式 hero。

## 6. 响应式规则

桌面：
- 左上 logo，右上导航。
- 杂志流为两列大图。
- 档案/事件为横向索引表。
- 关于页为双栏编辑版式。

平板：
- 只要封面宽度仍然可读，就优先保留双列。
- 先减少横向间距，再缩小图片尺度。
- 导航当前态继续使用细描边。

移动：
- 导航折叠为菜单按钮。
- 杂志流变为单列。
- 图片可以右移或局部溢出。
- 荧光绿符号可以继续遮挡，但不能影响主要阅读。
- 档案/事件行在宽度不足时把元信息折到标题下方。
- 关于页先联系方式，后机构介绍。

## 7. 使用检查表

应用这个系统时检查：
- 页面是否主要由近白和深灰构成？
- 荧光绿是否只是尖锐的符号强调，而不是泛滥成主题色？
- 是否避免了卡片、阴影和圆角容器？
- 桌面首屏是否保留了足够空白？
- 杂志页是否以图像为主，而不是以文字卡片为主？
- 档案/事件页是否像清爽的印刷目录？
- 字体是否基本保持中黑无衬线？
- 移动端是否保留实验性叠加，同时仍然可读？

## 8. 边界

不要复制：
- 原站商用字体文件。
- 原站杂志封面资产。
- 原站品牌 SVG 或动态图形。
- 原站源代码和动效文件。

可以复用：
- 近白档案表面。
- 紧排中黑无衬线字体。
- 大幅编辑图像网格。
- 细描边导航状态。
- 线性分隔的索引列表。
- 荧光绿符号叠加的品牌手势。
